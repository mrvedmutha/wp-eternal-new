## 1. Mission Statement

Scaffold a reusable, typographic Gutenberg block (`wp-rig/editorial-wide-text-section`) that renders centred, full-width editorial text in one of three layout variants. The variant controls both the content fields shown and the vertical rhythm of the section. No animation, no sticky behaviour — pure typography and spacing.

  "variant": {

## 5. Semantic Markup

```html
<!-- heading-only -->
<section class="ewts ewts--heading-only">
  <div class="ewts__inner">
    <h2 class="ewts__heading">{headingText}</h2>
  </div>
</section>

<!-- pull-quote -->
<section class="ewts ewts--pull-quote">
  <div class="ewts__inner">
    <blockquote class="ewts__quote">{quoteText}</blockquote>
  </div>
</section>

<!-- heading-body -->
<section class="ewts ewts--heading-body">
  <div class="ewts__inner">
    <h2 class="ewts__heading">{headingText}</h2>
    <p class="ewts__body">{bodyText}</p>
  </div>
</section>
```

> Pull-quote uses `<blockquote>` (not `<h2>`) for semantic accuracy and accessibility.

## 6. Editor UI (edit.js)

InspectorControls panel "Editorial Wide Text Section":

1. **Variant** — `SelectControl` with options: Heading Only / Pull Quote / Heading + Body
2. **Heading** — `TextareaControl` (shown when variant is `heading-only` or `heading-body`)
3. **Quote** — `TextareaControl` (shown when variant is `pull-quote`)
4. **Body** — `TextareaControl` (shown when variant is `heading-body`)

Canvas preview renders the active fields with correct BEM class names.

## 8. User Stories

- As a content editor, I can choose between three layout variants via a dropdown in the block sidebar.
- As a content editor, I only see the fields relevant to my chosen variant — no confusion.
- As a site visitor, I see the correct typographic treatment (upright heading, italic quote, or heading+body) based on the variant.
- As a site visitor on any device, the block is readable and correctly spaced.
